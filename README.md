# AI Crypto Advisor

A personalized crypto investor dashboard built with Next.js. The app learns user preferences through an onboarding quiz, then serves daily AI-curated insights, live market data, news, and memes — all tailored to the user's investor profile.

**Live App:** [https://crypto-advisor-PLACEHOLDER.vercel.app](https://crypto-advisor-PLACEHOLDER.vercel.app)
**Repository:** [https://github.com/PilPara/CryptoAdvisor](https://github.com/PilPara/CryptoAdvisor)

## Features

- **Authentication** — Email/password signup and login via Clerk
- **Onboarding Quiz** — Choose tracked assets, investor type, and content preferences
- **Coin Prices** — Live prices and 24h changes from CoinGecko
- **Market News** — Crypto news filtered by tracked assets via CryptoCompare
- **AI Insight of the Day** — Personalized daily insight generated by LLM (OpenRouter primary, Hugging Face fallback)
- **Crypto Memes** — Fresh memes from Reddit crypto subreddits (Imgflip fallback)
- **Voting System** — Thumbs up/down on every section, stored in DB for future model improvements

## Getting Started

### Prerequisites

- Node.js 18+
- A PostgreSQL database (e.g. [Neon](https://neon.tech/) free tier)
- API keys for Clerk, CoinGecko, OpenRouter, and Hugging Face (all free tier)

### Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/PilPara/CryptoAdvisor.git
   cd CryptoAdvisor
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Create a `.env` file based on `.env.example`:
   ```bash
   cp .env.example .env
   ```

4. Fill in your environment variables (see `.env.example` for all required keys).

5. Run the development server:
   ```bash
   npm run dev
   ```

6. Open [http://localhost:3000](http://localhost:3000).

## Database Access

The app uses **PostgreSQL hosted on [Neon](https://neon.tech/)**. The schema is managed with Drizzle ORM.

**Tables:**
- `users` — Maps Clerk auth IDs to internal user records
- `preferences` — Stores onboarding answers (tracked assets, investor type, content types)
- `votes` — Stores thumbs up/down feedback per section and content item

To inspect the database, connect using the `DATABASE_URL` from your environment variables with any PostgreSQL client (e.g. `psql`, TablePlus, or the Neon dashboard).
